## Interaction Contract

Since Phase 1 has no traditional UI, the "interaction contract" describes how users interact with the system's deliverables:

| Interaction | Mechanism | Validation |
|-------------|-----------|------------|
| Invoke researcher agent | User types `@researcher` in Claude Code | Success criterion 2: researcher completes work and returns results |
| Invoke writer agent | User types `@writer` in Claude Code | Success criterion 3: writer produces output with voice profile awareness |
| Memory read at task start | Agent explicitly Reads full MEMORY.md | Success criterion 4: memory lifecycle validated |
| Memory write at task end | Agent appends timestamped entries to MEMORY.md | Success criterion 4: structured append with timestamps |
| Run smoke tests | `node tests/smoke-test-paths.js` | Success criterion 5: all tests pass, exit code 0 |
| View agent reference | Read CLAUDE.md agent reference table | Success criterion 1: orchestrator loads with routing table visible |

**Human checkpoints in Phase 1:** Instruction-based only (per CONTEXT.md D-03). CLAUDE.md contains text like "After /strategy, present topic briefs and WAIT." No hook enforcement.
