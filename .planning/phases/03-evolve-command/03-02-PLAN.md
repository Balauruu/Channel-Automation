**Body structure -- numbered procedural steps (8 steps total):**

**Title:** `# Evolve`

**Preamble paragraph:** Brief description -- "Run this command after pipeline sessions to promote learned insights from Pending Review to Permanent memory. Dispatches the @observer to analyze new events, auto-promotes all pending entries, then shows a summary with optional revert."

**Step 1: Dispatch Observer (per D-05)**
- Use the Agent tool to dispatch `@observer` with prompt:
  ```
  Process new events. Read your cursor and process new runs from obs.jsonl. Report your completion summary when done.
  ```
- After observer completes, parse its text output for key stats using these regex patterns:
  - `Runs processed: (\d+)`
  - `Entries written: (\d+)`
  - `Candidates rejected: (\d+)`
- If observer dispatch fails (error from Agent tool), log the failure and continue to Step 2 -- observer failure must not block promotion of already-written entries (per RESEARCH.md Open Question 3).

**Step 2: Scan for Pending Entries (per D-06 early exit)**
- Run via Bash tool: `node .claude/scripts/memory/evolve.js scan`
- Parse the JSON output.
- If scan `total` is 0 AND observer reported 0 runs processed (or observer failed): display `"No new events. No pending entries. Nothing to do."` and STOP (per D-06).
- If scan `total` is 0 but observer wrote entries: re-run scan (observer just wrote them).

**Step 3: Promote All Entries (per D-02, D-03)**
- Run via Bash tool: `node .claude/scripts/memory/evolve.js promote`
- Parse the JSON output. Store the `promoted` array for display.

**Step 4: Display Summary (per D-01, D-02, D-08, EVLV-02)**
- Display observer stats first (if available):
  ```
  Observer processed N runs, wrote N entries, rejected N.
  ```
- Then display promoted entries grouped by type using this exact format:
  ```
  All entries auto-promoted to Permanent sections:

  **insights.md files:**
    1. [skill-name] [CONF] insight text
    2. [skill-name] [CONF] insight text

  **MEMORY.md files:**
    3. [agent-name] [CONF] insight text
    4. [agent-name] [CONF] insight text

  **PLAYBOOK.md:**
    5. [CONF] insight text
  ```
- Extract the skill/agent name from the file path (parent directory name).
- Extract confidence tag and insight text from the `promoted` field in the JSON.
- If `permanent_sections_created > 0`, note: `"(Created ## Permanent section in N files)"`

**Step 5: Revert Prompt (per D-09, D-10, EVLV-03)**
- Display: `"Revert any? (enter numbers, or press Enter to keep all)"`
- Wait for user input.
- If user presses Enter (empty input): proceed to Step 7.
- If user provides numbers (e.g., "2,4" or "2 4"): parse into integer list, proceed to Step 6.

**Step 6: Execute Reverts (per D-09)**
- Run via Bash tool: `node .claude/scripts/memory/evolve.js revert <n1> <n2> ...`
  (space-separated indices from user input)
- Parse the JSON output.
- Display what was reverted:
  ```
  Reverted N entries:
    - #2: [agent-name] insight text
    - #4: [agent-name] insight text
  ```

**Step 7: Commit Changes**
- Stage all modified memory files using `git add` with specific file paths from the promote output (NOT `git add -A` -- per project git rules).
- Commit with message: `evolve: promote N entries to permanent memory`
- If reverts happened, the commit message should be: `evolve: promote N entries, revert M entries`

**Step 8: Done**
- Display: `"Evolution complete. N entries promoted, M reverted."`

**Important constraints to embed in the skill body:**
- "Do NOT prompt the user per-entry. Auto-promote ALL entries, then offer batch revert." (per D-02/D-04)
- "Do NOT use the Edit tool for bulk file operations. All file manipulation goes through evolve.js." (per RESEARCH.md anti-patterns)
- "If observer fails, still promote existing Pending Review entries." (per RESEARCH.md Open Question 3)
