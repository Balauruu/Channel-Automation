# Insights

Accumulated learnings from skill runs. Append general insights only — no project-specific findings.

- When the Write tool is blocked for report files, Python script via Bash tool is a reliable fallback for writing structured markdown outputs to disk
- Narration word count (cues stripped) is the correct measure for runtime estimation -- always run the word count script early in drafting, not after the full script is written; first draft was 3,075 words against a 5,250+ target, requiring a full rewrite
- Pipeline status reporting works well as a checkpoint summary in chat before each stage -- a table of channel/video counts gives the user enough signal to abort before analysis commits to potentially wrong data
